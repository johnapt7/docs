# Boku Adapter

The Boku Adapter enables mobile payment processing through carrier billing and direct mobile payments across global networks.

## Key Features

- **Carrier Billing**: Direct billing to mobile phone accounts
- **Global Coverage**: Support for 65+ countries
- **One-Click Payments**: Simplified mobile checkout
- **Real-time Processing**: Instant payment confirmations

## Getting Started

### Prerequisites

Before integrating the Boku adapter, ensure you have:

- Boku merchant account
- API credentials (Merchant ID and Security Key)
- Approved service configuration
- Webhook endpoints configured

### Basic Configuration

```json
{
  "adapter": "boku",
  "config": {
    "merchantId": "your-merchant-id",
    "securityKey": "your-security-key",
    "environment": "production",
    "serviceId": "your-service-id"
  }
}
```

## Integration Steps

1. **Configure Credentials**: Set up your Boku API credentials
2. **Initialize Adapter**: Create an instance of the Boku adapter
3. **Implement Flows**: Add support for mobile payment flows
4. **Set Up Webhooks**: Configure webhook handlers for notifications
5. **Test Integration**: Test with test carriers in sandbox

## Security Considerations

<Warning>
Always validate phone numbers and implement fraud prevention measures. Store API credentials securely and never expose them in client-side code.
</Warning>

## Rate Limits

Boku enforces the following rate limits:
- **API Calls**: 60 requests per minute
- **Bulk Operations**: 10 concurrent requests
- **Webhook Deliveries**: 30 per second per endpoint

<Note>
Rate limits may vary based on your merchant agreement. Contact Boku support for higher limits.
</Note>
